# Component Catalog — Raw Reference

**Platform:** RN Game Studio (Real Nation Digital)  
**Status:** Factual baseline for rollout planning and sales collateral  
**Last updated:** July 2026  
**Sources:** `packages/shared/src/module-registry.ts`, `measurement/registry.ts`, `COMPONENT_DATA_EVENT_INVENTORY.md`, player/runtime code

Use this document as **raw material** for spreadsheets: master table (§2), then per-type detail (§3–§5). Effort bands in §2 are **indicative designer setup time** for a new client instance from Studio defaults — not engineering build time.

**Legend**

| Tag | Meaning |
|-----|---------|
| **Shipped** | In production; Studio editor + public player |
| **Planned** | In registry; not participant-facing yet |
| **Emitted** | Event sent to `POST /api/track` today |
| **Canonical** | Target name in measurement registry; may map from legacy |
| **Legacy** | Still emitted; mapped to canonical on ingest |

## 3. Foundation shells — detail

### 3.1 Course (`course`)

**What it is**  
Structured learning or participation programme. Learners enrol via open link or class code, progress through sections/items, resume via learning link email, earn badges/certificates shown on course home.

**Customisable elements**

| Area | Elements |
|------|----------|
| Metadata | Title, description, slug, client/project/design codes, thumbnail, status (draft/published/archived) |
| Presentation | Backgrounds (desktop/tablet/mobile), logo, colours (headline/body/accent/card), favicon, powered-by, loading text |
| Settings | Navigation mode (sequential/free), layout (rows/cards/bento), learning link labels + GDPR acknowledgement, class code (enrollment mode parked) |
| Structure | Sections (title, description, icon, unlock date/days), items: **module**, **experience**, or **video** |
| Items | Display title, icon, lock-after-complete, link to module instance or experience |
| Measurement | Deployment-level M&R panel (tracking, retention, reporting toggles) |

**Trackable events (emitted today)**

| Event | When |
|-------|------|
| `course.viewed` | Course home shown |
| `course.enrolled` | New session created |
| `course.resumed` | Session resumed via token |
| `course.started` | First home view after new enrolment |
| `course.item_started` | Learner opens an item |
| `course.item_completed` | Item marked complete |
| `course.completed` | All items complete |

**Data stored (session blob)**  
`sessionId`, `courseId`, `participantId`, `email?`, `resumeToken`, `completedItemIds`, `currentItemId`, `earnedCertificates`, `earnedBadges`, `outcomes`, `itemOutcomes`, `data` (form fields, learner display name), `enrolledViaClass?`

**Important notes**

- Learning link stores **email** for resume lookup.
- Class code validated but **not stored verbatim** on session.
- Course context on child `track()` events is **improving** but not all legacy events carry `courseSessionId` — join via session blobs for reporting.
- Video items: URL only — no dedicated player analytics.
- Educator profile panel: **parked** in schema.

**Rollout / sales**

- Sell as: digital programme, onboarding path, class-based learning, badge/certificate journey.
- Effort driver: number of sections, nested flows, form/survey items, branding depth.

---

### 3.2 Flow / Experience (`flow`)

**What it is**  
Ordered (or graph) assembly of components with shared session, outcome merging, and experience-level navigation. React Flow canvas in Studio; linear execution today with logic stub passthrough.

**Customisable elements**

| Area | Elements |
|------|----------|
| Metadata | Title, slug, client/project/design codes, thumbnail, notes, template |
| Graph | Module nodes (reference to configured instances), control nodes (entry/exit/logic/join/delay/redirect) |
| Per-node overrides | Completion behaviour, end-screen copy/CTAs, leaderboard mode in-flow |
| Foundation | Tracking/reporting toggles, session TTL, navigation (back button policy), kiosk idle timeout |
| Measurement | Deployment-level M&R panel |

**Trackable events (emitted today)**

| Event | When |
|-------|------|
| `flow.viewed` | Flow player loaded |
| `flow.started` | New flow session |
| `flow.resumed` | Returning session |
| `flow.step_started` | Step iframe loaded |
| `flow.step_completed` | Child step finished |
| `flow.completed` | All steps done |

**Legacy aliases (still mapped on ingest)**  
`experience.step_start` → `flow.step_started`, `experience.step_complete` → `flow.step_completed`, `experience.complete` → `flow.completed`

**Data stored**  
`experience-session:{id}` — `participantId`, `currentStepIndex`, `history`, `outcomes`, `data`, timestamps

**Important notes**

- Branding lives on **child components**, not the experience shell.
- Quiz (play-along), pinboard, leaderboard **not embeddable** in flows today.
- Wave 4 branching (logic/join rules) **paused** — graph exists, logic is stub.
- Draft preview uses `previewToken` in URL.

**Rollout / sales**

- Sell as: campaign journey, multi-step engagement, lead nurture path, event activation flow.
- Effort driver: step count, component variety, override copy per step.

## 4. Shipped components — detail

### 4.1 Spinning wheel (`spinning-wheel`)

| | |
|-|-|
| **Description** | Branded prize wheel; weighted segments; win/lose panels and sounds |
| **Category** | Game |
| **Standalone URL** | `/{slug}` |
| **In flow / course** | Yes / Yes |

**Customisable elements**  
Segments (labels, weights, win/lose per segment), wheel/frame/background assets, logo, headline, spin button, result panels (incl. per-segment panel images), sounds (spin, segment reveal, music), spin animation (rotations, duration, easing), landscape gate, favicon, thumbnail, reporting toggle → Google Sheet tab, powered-by

**Trackable events**

| Event | Emitted | Notes |
|-------|---------|-------|
| `wheel.spin` | Yes | segmentIndex, prizeLabel, outcome |
| `page.step_complete` | Yes (in flow) | segmentId, segmentLabel, isWin, completed |
| `component.viewed` | Canonical | Not emitted yet |
| `component.completed` | Canonical | Use page.step_complete today |

**Optional:** `POST /api/log-spin` → Google Sheets when `reportingEnabled`

**Data / PII**  
Segment outcomes only. No player identity.

**Important notes**  
Most mature **reporting** path (Sheets + `{slug}_Report` SPA). Landscape-oriented. Works in course embed.

| | |

### 4.3 Flip cards (`flip-cards`)

| | |
|-|-|
| **Description** | Deck of cards with front/back content; engagement / memory style |

**Customisable elements**  
Per-card: front/back images, header, body, overlay button text, optional sound. Global: shuffle, mute, fullscreen toggles, fonts, colours, background, reporting toggle

**Trackable events**  
`flip_cards.open` (per flip), `page.step_complete` (flipCards.engaged, completed)

**Important notes**

### 4.4 Quiz — Play-Along (`quiz`)

| | |
|-|-|
| **Description** | Live synchronized quiz: host, presenter, join, kiosk surfaces |
| **Standalone URLs** | `/quiz/{slug}/host`, `/join`, `/present`, `/kiosk`, `/live/{code}/leaderboard` |

**Customisable elements**  
Mode (play-along), branding (colours, logos, fonts), tracks/questions, play-along settings (max participants, retention, profanity filter, bonus scoring), reporting toggle

**Trackable events**  
`quiz.answer` (participantId, choiceId), room events in `quizsession:{CODE}` blob (not all in track)

**Data / PII**  
Participant display names, scores, answers in session blob. Host key for control.

**Important notes**  
**Not embeddable** in flows or courses. Separate product surface from mini-quiz. Room sessions expire (`expiresAt`). Kiosk/single-player only for in-flow use per roadmap.

|-|-|
| **Description** | Collaborative wall: guest submit stickies/photos; live board; moderation |
| **URLs** | `/pinboard/{slug}`, `/submit`, `/moderate` |

**Customisable elements**

### 4.7 Runner (`runner`)

| | |
|-|-|
| **Description** | Side-scrolling endless runner; up to 8 characters; parallax |

**Customisable elements**  
Characters (run/jump/death sprite sheets), parallax layers, ground, items (+/-), gameplay tuning, HUD, intro/end, fullscreen, linked leaderboard, fonts — **largest editor surface**

**Trackable events**  
`runner.round_start`, `runner.round_end` (score), `page.step_complete` (score/distance/time)

### 4.8 Leaderboard (`leaderboard`)

| | |
|-|-|
| **Description** | Live score display; linked from catch/runner or manual entry |

**Customisable elements**  
Mode (linked/manual), linked game reference, moderator PIN, board + moderator branding

**Trackable events**  
`leaderboard.view` only — scores via `POST /api/leaderboard-state`

`displayName` (user-entered), `externalId` (pseudonymous)

### 4.9 Landing page (`landing`)

| | |
|-|-|
| **Description** | Flexible marketing/intro page with block editor |

**Customisable elements**  
Shared page chrome: logo, backgrounds, typography, headline/body, primary CTA. **Screens** with blocks: text, image, image+text, gallery, video, spacer, divider, button, embed/iframe, poll. Page settings: width, alignment, padding, entrance animation. Flow end-screen flag per screen.

**Trackable events**  
`page.step_complete` (landing.cta, completed). Poll votes → `poll-state` API (not track)

### 4.10 Form (`form`)

| | |
|-|-|
| **Description** | Configurable multi-field data capture (survey, registration) |

**Customisable elements**  
Field types: text, email, phone, postcode, date, dropdown, multiple choice, checkbox. Per field: label, placeholder, required, options. Submit label, post-submit screen, page chrome, reporting toggle

**Trackable events**  
`page.step_complete` with `form.fieldValues`. Canonical: `form.submitted` (not separate yet)

**Data / PII**

**Important notes**  
CSV export in Studio. Feeds certificate/badge merge fields. Duplicate storage paths (`form.fieldValues` vs `data.formFields`).
